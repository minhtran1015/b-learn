# Bài 1. Mô hình hóa dữ liệu trong hệ thống Big Data

Trong Big Data, dữ liệu thô hiếm khi sẵn sàng để phân tích ngay. Muốn khai thác giá trị, dữ liệu phải được mô hình hóa, làm sạch và liên kết.

## 1. Tại sao cần mô hình hóa dữ liệu
- Dữ liệu thường phân tán ở nhiều hệ thống khác nhau.
- Dữ liệu có thể kém chất lượng, thiếu giá trị hoặc có giá trị sai lệch.
- Nhiều mô hình học máy cần đặc trưng mà dữ liệu gốc chưa có sẵn.

## 2. Quá trình chuyển đổi dữ liệu
- Raw data như hình ảnh, gen, văn bản.
- Trích xuất đặc trưng và làm sạch dữ liệu.
- Biến dữ liệu thành structured data có thể dùng cho phân tích và học máy.
- Các quan hệ cấu trúc đôi khi chính là đặc trưng quan trọng nhất.

## 3. Một số hướng mô hình hóa phổ biến
- Linked Data: tìm mẫu thông qua các kết nối, cụm và đường đi.
- Knowledge Graphs: biểu diễn lớp, lớp con, thực thể và thuộc tính.
- Dynamic Data: theo dõi sự thay đổi theo thời gian để dự báo.
- Tabular Data: dữ liệu bảng kết hợp với lookup hoặc join từ hệ thống ngoài.

## 4. Kết luận
Mô hình hóa dữ liệu là bước nền tảng để biến dữ liệu rời rạc thành dữ liệu có thể phân tích, truy vấn và học máy.
